Artifact 3 · Research Report

Machine Learning vs. Deep Learning: Email Spam Filtering and Facial Recognition

This artifact is a research-style report that explains the differences between machine learning (ML) and deep learning (DL) by analyzing two real-world artificial intelligence applications: email spam filtering and facial recognition. The report demonstrates when traditional machine learning algorithms are more effective and when deep learning approaches are required for complex, high-dimensional problems.

Introduction: Machine Learning and Deep Learning are core technologies powering modern artificial intelligence. While both approaches enable computers to learn from data, they differ in architecture, data requirements, feature extraction methods, computational needs, and explainability. This report compares a machine learning-based spam filtering system with a deep learning-based facial recognition system.

Objective: To explain the practical differences between machine learning and deep learning by evaluating two real-world applications and identifying why each technology is suitable for specific types of problems.

Description: The first case study focuses on email spam classification, using Gmail's spam filtering system as an example of how traditional machine learning algorithms such as Naive Bayes, Logistic Regression, and Support Vector Machines (SVM) can effectively classify structured text data. The second case study focuses on facial recognition technologies such as Apple Face ID, demonstrating how deep learning models, especially Convolutional Neural Networks (CNNs), process complex image data.

Machine Learning Case Study - Email Spam Filtering: Traditional machine learning is highly effective for spam detection because email data contains structured and interpretable features. Algorithms can analyze word frequency, suspicious links, sender information, and other measurable patterns to classify messages as spam or legitimate. ML models provide fast processing, require less computational power, and offer better explainability compared to deep learning approaches.

Why Deep Learning is Not Ideal for Spam Filtering: Applying deep learning to basic spam detection can introduce unnecessary complexity. Deep neural networks require significantly more computational resources, create explainability challenges, and may increase operational costs without providing meaningful improvements for problems that already have clearly defined features.

Deep Learning Case Study - Facial Recognition: Facial recognition requires processing large amounts of unstructured visual data consisting of millions of pixel values. Deep learning models, particularly Convolutional Neural Networks (CNNs), are designed for this type of problem because they automatically learn important facial features such as edges, shapes, and complex patterns without requiring manual feature engineering.

Why Machine Learning is Not Ideal for Facial Recognition: Traditional machine learning struggles with facial recognition because it depends heavily on manually created features. Changes in lighting, facial expressions, angles, and appearance can significantly reduce accuracy. Deep learning models overcome these limitations by learning complex spatial relationships and adapting to variations in real-world environments.

Process: I researched machine learning algorithms, deep learning architectures, spam classification methods, facial recognition technologies, and AI model comparison frameworks. I analyzed both applications based on data type, feature extraction, hardware requirements, model transparency, scalability, and business impact.

Tools and Technologies Used: Microsoft Word, Brightspace course materials, IBM AI/ML resources, Apple Face ID documentation, Gmail spam filtering concepts, academic research papers, ChatGPT, and Claude.

Value Proposition: This artifact demonstrates the ability to evaluate AI technologies based on real-world requirements instead of simply comparing theoretical definitions. It highlights how selecting the correct AI approach improves efficiency, accuracy, scalability, and business outcomes.

Unique Value: The report provides a balanced comparison by showing that deep learning is not always the best solution. It explains why simpler machine learning models can outperform complex neural networks when the problem involves structured and interpretable data.

Relevance: Understanding the differences between ML and DL is essential for modern AI decision-making. Organizations must select models based on data complexity, computational resources, explainability requirements, and expected business value.

References: Goodfellow et al. (2016), LeCun et al. (2015), Sahami et al. (1998), Viola & Jones (2001), Apple Face ID documentation, IBM AI resources, and Indiana Wesleyan University course materials.
